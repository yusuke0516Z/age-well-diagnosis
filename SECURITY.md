# 🔐 Age-Well 診断システム - セキュリティガイド

## ⚠️ 緊急セキュリティパッチ適用済み

このリポジトリには**重大なセキュリティ脆弱性**が発見され、緊急で修正されました。

---

## 🚨 発見された脆弱性（修正済み）

### 1. **認証情報のハードコード（Critical）**
- **問題**: 管理者IDとパスワードが平文でソースコードに記載
- **影響**: 誰でもソースコードを見て管理画面にアクセス可能
- **修正**: ハッシュ値による認証に変更、設定ファイル分離

### 2. **クライアント側のみの認証（Critical）**
- **問題**: JavaScriptコンソールで認証をバイパス可能
- **影響**: セッション管理の脆弱性
- **修正**: セッション管理の強化、タイムアウト実装

### 3. **XSS脆弱性（High）**
- **問題**: innerHTML によるスクリプトインジェクション
- **影響**: ユーザーデータの窃取、セッションハイジャック
- **修正**: Security.escapeHtml() によるサニタイゼーション

### 4. **CSRF対策なし（High）**
- **問題**: クロスサイトリクエストフォージェリ対策が不十分
- **影響**: 不正なデータ送信
- **修正**: トークン検証の推奨

### 5. **GAS URL の公開（Medium）**
- **問題**: Google Apps Script URL がソースコードに記載
- **影響**: 直接アクセスによるデータ不正取得
- **修正**: config.js に分離、GAS側でのアクセス制限推奨

---

## ✅ 適用された対策

### 1. セキュアな認証システム
- **SHA-256ハッシュ**: パスワードはハッシュ値で保存
- **ログイン試行制限**: 5回失敗で15分間ロック
- **セッション管理**: 1時間の自動タイムアウト
- **入力サニタイゼーション**: すべての入力値を検証

### 2. XSS対策
- `Security.escapeHtml()`: すべてのユーザー入力をエスケープ
- `Security.sanitizeInput()`: 危険なタグとスクリプトを除去
- `textContent` の使用推奨

### 3. HTTPS強制
- 本番環境で自動的にHTTPSにリダイレクト
- ローカル開発環境 (localhost) は除外

### 4. 設定の分離
- `config.js`: 機密情報を含む設定ファイル
- `.gitignore`: config.js をGitから除外
- `config.example.js`: サンプル設定

## 📋 セットアップ手順（必須）

### ステップ1: セキュリティ設定の生成

1. `setup-security.html` をブラウザで開く
2. 管理者メールアドレスを入力
3. **強力なパスワード**を入力（12文字以上、英数字記号混在）
4. Google Apps Script URL を入力（デプロイ後）
5. 「config.js 生成」ボタンをクリック
6. 生成された内容をコピー
7. `config.js` ファイルに貼り付けて保存

### ステップ2: .gitignore の確認

```bash
# config.js が .gitignore に含まれていることを確認
cat .gitignore | grep config.js
```

**絶対に** config.js を Git にコミットしないでください。

### ステップ3: HTTPS設定（本番環境）

本番環境では必ず HTTPS を使用してください:

```javascript
// config.js で enforceHttps を true に設定
SECURITY: {
  enforceHttps: true,  // 本番環境では必須
}
```

### ステップ4: Google Apps Script の保護

GASのデプロイ設定:
1. Apps Script エディタで「デプロイ」→「デプロイを管理」
2. 「アクセスできるユーザー」を「自分のみ」に設定
3. または、GAS側でトークン検証を実装

## 🛡️ セキュリティベストプラクティス

### 管理者向け

1. **強力なパスワード**
   - 最低12文字以上
   - 大文字・小文字・数字・記号を含む
   - 辞書にない単語
   - 他のサービスと使い回さない

2. **定期的なパスワード変更**
   - 3〜6ヶ月ごとに変更
   - setup-security.html で再生成

3. **アクセスログの監視**
   - 不審なログイン試行をチェック
   - GASのログを定期確認

4. **HTTPS必須**
   - HTTP接続は絶対に使用しない
   - Let's Encrypt などで無料SSL証明書取得

### 開発者向け

1. **config.js は絶対にコミットしない**
   ```bash
   # 誤ってコミットした場合
   git rm --cached config.js
   git commit -m "Remove config.js from repository"
   ```

2. **入力値は必ずサニタイズ**
   ```javascript
   // ❌ 危険
   element.innerHTML = userInput;

   // ✅ 安全
   element.textContent = Security.escapeHtml(userInput);
   ```

3. **サーバーサイド認証への移行**
   - クライアント側認証は一時的な措置
   - 本番環境では Firebase Auth、Auth0、または独自API推奨

## 📞 セキュリティインシデント発生時

### 即座に実施すること

1. **パスワード変更**
   - setup-security.html で新しいハッシュ値を生成
   - すべての管理者に通知

2. **セッションクリア**
   - ブラウザのlocalStorageを削除
   - 全ユーザーに再ログインを要求

3. **ログ確認**
   - GASのログで不審なアクセスを確認
   - IPアドレスと時刻を記録

4. **システム停止の検討**
   - 深刻な場合は一時的にサービス停止
   - 調査完了まで再開しない
